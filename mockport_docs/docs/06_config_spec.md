# 06. Configuration Spec

## File name

Default:

```txt
mockport.yml
```

Docker path:

```txt
/etc/mockport/mockport.yml
```

## Minimal config

```yaml
version: "0.1"

server:
  host: 127.0.0.1
  port: 43101

mode: ai-safe

adapters:
  stripe:
    enabled: true
    base_path: /stripe
    scenario: payment_success
    fake_secret: mockport_stripe_secret
    webhook:
      target_url: http://app:3000/webhooks/stripe
      signing_secret: whsec_mockport
```

## Server config

```yaml
server:
  host: 127.0.0.1
  port: 43101
```

## Mode

Supported modes:

```txt
ai-safe
local
strict
```

### ai-safe

Default. Warns or blocks dangerous settings.

### local

Allows local developer flexibility but still redacts secrets.

### strict

Fails startup if real-looking secrets or real service URLs are detected.

## Adapter config

```yaml
adapters:
  stripe:
    enabled: true
    base_path: /stripe
    scenario: payment_success
    fake_secret: mockport_stripe_secret
```

## Scenario config

Initial simple form:

```yaml
scenarios:
  stripe_payment_success:
    adapter: stripe
    match:
      method: POST
      path: /v1/checkout/sessions
    response:
      status: 200
      body:
        id: cs_test_mockport
        object: checkout.session
        payment_status: paid
```

## Environment override

Use environment variables for runtime override:

```env
MOCKPORT_CONFIG=/etc/mockport/mockport.yml
MOCKPORT_PORT=43101
MOCKPORT_MODE=ai-safe
```

## Generated `.env.mockport`

```env
# Generated by Mockport. Safe for local and CI use.

STRIPE_API_URL=http://localhost:43101/stripe
STRIPE_SECRET_KEY=mockport_stripe_secret
STRIPE_WEBHOOK_SECRET=whsec_mockport
```

## Redaction rule

Never print full secret values.

Allowed output:

```txt
STRIPE_SECRET_KEY=mockport_...cret
```

For real-looking secrets, show only key name and category:

```txt
STRIPE_SECRET_KEY looks like a real Stripe live key.
```
